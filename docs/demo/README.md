## 무엇을 보여주나

어드민 콘솔의 **고객 조회** 화면을, 직원 3명이 **서로 다른 역할로 차례로 로그인**해서 엽니다. 같은 고객 목록이 **역할에 따라 다르게 보이고**, 민감정보 조회는 **사유와 함께 감사로그에 영구 기록**되는 걸 한 흐름으로 시연합니다. → 인증보안계의 *"신원 → 권한 → 감사"* 서사.

## 보는 포인트

- **권한(역할)에 따라 같은 데이터가 다르게** 보인다 — 마스킹은 자동, 창구는 사유 필수.
- 민감정보 접근은 **`customer_access_log`에 영구 기록** ("모든 조회 이력은 영구 적재됩니다").
- 역할은 게이트웨이가 **JWT에서 추출·전파한 `X-User-Role`** 기반.
