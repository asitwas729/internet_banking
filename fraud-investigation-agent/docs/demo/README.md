# 데모 영상 — 이상거래 조사 에이전트 콘솔

**파일**: [`fraud_agent_demo.webm`](fraud_agent_demo.webm) (약 27초) · 스크린샷: `console_*.png`

> ⚠️ GitHub 마크다운에선 `.webm`이 자동재생되지 않습니다. 파일을 내려받아 재생하거나, 아래 **장면 설명 + 스크린샷**으로 내용을 파악하세요.

## 무엇을 보여주나

어드민 콘솔(`/admin/fraud`)에서 분석가가 **이상거래 조사 큐의 사건을 클릭**하면, 에이전트가 **조사 루프**(가설 경합 → 도구 선택 → 증거 반영 → 권고)를 돌려 **단계별 트레이스와 권고**를 한 화면에 그려주는 모습입니다. 사건 4개를 차례로 눌러, **같은 큐에서 사건마다 다른 결론·다른 경로**가 나오는 걸 보여줍니다.

## 장면 순서 (사건 4개를 차례로 클릭)

| # | 사건 | 에이전트가 보여주는 것 |
|---|---|---|
| 1 | **계정탈취** (ALT-H2) | 낯선 기기 → 인증 조사 → **인증실패 신호로 H2(계정탈취) 확정**. 인증 단계는 실제 customer-service(인증보안계)를 호출해 **"● 실연결" 배지** 표시 |
| 2 | **사망계좌** (ALT-DEATH) | `get_party`에서 사망 확인 → **즉시 fail-closed**, 헤드라인 "사망계좌 · 권리자 적격성(L4)" (이상도 낮아도 결정적 사실로 종료) |
| 3 | **경로 분기** (ALT-H1-FLIP) | 보이스피싱처럼 보이지만 낯선 기기 + 인증실패로 **H2(계정탈취)로 뒤집힘** — "결과가 다음 행동을 바꾼다" |
| 4 | **정상/오탐** (ALT-H5) | 수취망·AML·STR 깨끗 → **BENIGN(정상)**, 권고 동작 불요 |

## 보는 포인트

- 5개 가설이 **균등 분포에서 출발** → 도구를 *선택 이유와 함께* 호출 → **증거로 분포 재계산** → 확정/종료.
- 같은 조사 큐에서 **사건마다 결론도 경로도 다르다** (고정 워크플로우 아님).
- 실제 조치(지급정지·STR)는 **분석가 승인(HITL) + RBAC** 통과 시에만 — 에이전트는 권고까지만.
